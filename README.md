# media_service

Небольшой сервис-посредник для фильтрации изображений через YOLO-модель перед отправкой в downstream-сервис.

## Что делает сервис

1. Принимает изображение через HTTP (`POST /upload`)
2. Прогоняет его через локальную YOLO-модель `cvModel.pt`
3. Смотрит на максимальный confidence среди найденных объектов
4. Если модель **не уверена** (`confidence < 0.5`) — пересылает изображение дальше во внешний обработчик
5. Если модель уверена — помечает изображение как отфильтрованное и дальше не отправляет

Идея простая: сервис стоит между источником изображений и основным обработчиком и отбрасывает то, что модель считает достаточно уверенным кейсом.

---

## Как работает пайплайн

```text
client/test_script -> POST /upload -> YOLO inference
                                      ├─ confidence >= 0.5 -> filtered
                                      └─ confidence < 0.5  -> forwarded to target service
```

## API

### `POST /upload`

Принимает файл изображения в multipart/form-data.

#### Пример запроса

```bash
curl -X POST "http://127.0.0.1:5000/upload" \
  -F "file=@image.jpg"
```

#### Возможные ответы

**Если изображение переслано дальше:**

```json
{
  "status": "forwarded",
  "yolo_conf": 0.23
}
```

**Если изображение отфильтровано:**

```json
{
  "status": "filtered",
  "yolo_conf": 0.87
}
```

**Если произошла ошибка при пересылке:**

```json
{
  "status": "error",
  "message": "..."
}
```

## Локальный запуск

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Распаковать тестовые данные

```bash
7z x val.7z -y
```

### 3. Запустить сервис

```bash
python media_service.py
```

Сервис стартует на:

```text
http://127.0.0.1:5000
```

### 4. Запустить тестовую отправку изображений

В другом терминале:

```bash
python test_script.py
```

---

## Запуск через Docker

### Сборка образа

```bash
docker build -t media-service .
```

### Запуск контейнера

```bash
docker run --rm -p 5000:5000 media-service
```

## Важные текущие настройки

Сейчас в коде захардкожены следующие значения:

- путь к модели: `cvModel.pt`
- адрес downstream-сервиса: `http://127.0.0.1:8000/eventProcessor/img`
- порог пересылки: `0.5`
- локальный порт FastAPI: `5000`
- папка с тестовыми изображениями в `test_script.py`: `media-Service/val`

Если проект будет развиваться дальше, это лучше вынести в переменные окружения.

## Ограничения текущей версии

- настройки захардкожены в коде
- нет логирования в нормальном виде
- нет healthcheck-эндпоинта
- нет обработки очередей/батчей
- нет отдельного конфига для dev/prod
- в Docker оба скрипта запускаются в одном контейнере через shell, что удобно для теста, но не очень хорошо для production
